# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Tech Stack

**ASP.NET Core 8 Web API** (C#, nullable + implicit usings enabled). It is the shared backend for the Novopharma inventory solution — consumed by `novo-inventaire-mobile` (field counting) and `novo-inventaire-dashboard` (admin). Persists to a **SQL Server** database (`NOVOPHARMA`) that also hosts a **SAGE ERP** schema (tables prefixed `F_…`).

> The solution is `novo-invnt-back.sln`; the project folder is `tunisair-back/` with root namespace `tunisair_back` (a leftover template name, same lineage as `tunisair-admin` on the dashboard).

## Commands

Run from the repository root (where `novo-invnt-back.sln` lives):

```bash
dotnet restore                         # Restore NuGet packages
dotnet build                           # Build the solution
dotnet run --project tunisair-back     # Run the API (Development by default)
```

- **HTTPS**: https://localhost:7099 — **HTTP**: http://localhost:5194
- **Swagger UI** (Development only): https://localhost:7099/swagger
- Switch environment with `ASPNETCORE_ENVIRONMENT` (`Development` / `Production`).

There is no test project in the solution.

## Configuration & secrets

Configuration lives in `tunisair-back/appsettings.json`:
- `ConnectionStrings:DefaultConnection` — SQL Server connection (server, DB, credentials).
- `Jwt:Key` / `Jwt:Issuer` / `Jwt:Audience` — JWT signing secret and claims.
- `EPPlus:ExcelPackage:LicenseContext` — set to `NonCommercial`.

⚠️ **`appsettings.json` and `appsettings.Development.json` are git-ignored** (they contain a real DB password and JWT key). After cloning, copy `appsettings.example.json` → `appsettings.json` and fill in real values. Never commit real secrets; prefer **User Secrets** (`dotnet user-secrets`) or environment variables for local dev.

## Architecture

### Database-first EF Core (do not hand-edit scaffolded files)
The data layer was **scaffolded from the existing SQL Server DB using EF Core Power Tools** (see `efpt.config.json`). `Models/DTHDLGContext.cs` and the SAGE entity classes (`FArticle`, `FDepot`, `FArtGamme`, `FArtEnumRef`, …) are marked `// <auto-generated>`. **Regenerate them via EF Power Tools rather than editing by hand** — manual edits are lost on the next scaffold. The SAGE entities map huge ERP tables (FArticle has ~200 columns) including audit triggers.

Two kinds of entities coexist in `Models/`:
- **SAGE (read-mostly, `F_` tables)**: `FArticle`, `FDepot`, `FArtGamme`, `FArtEnumRef`, `FDocLigne` — the ERP master data.
- **Application-owned (read/write)**: `User`, `Inventaire`, `ProduitInventaire`, `LigneInventaire`, `UserInventaire`, `UserDepot` — the inventory workflow tables.

### Controllers (`Controllers/`, all `api/[controller]`)
| Controller | Role |
|---|---|
| `AuthController` | Register, login (JWT), list/get users, change password, delete |
| `InventaireController` | **Core** — inventory CRUD, status, Excel import, counting (`modifier-quantite`), reports (`rapport1/2`, `rapportExcel`), error lines, final quantities, affectations |
| `ArticleController` | SAGE article lookup by ref/barcode/designation, `typeGestion` resolution, gamme & serial checks |
| `AffectationController` | Assign users to inventories (max 2) and depots to users |
| `FDepotController` | SAGE depot master data |
| `LigneInventaireController` | Alternative/legacy raw-count line interface (syncs into `ProduitInventaire`) |
| `FDocLigneController` | **Currently disabled** (endpoints commented out) — SAGE document lines |
| `FallbackController` | SPA catch-all → serves `wwwroot/index.html` |

### Authentication
JWT Bearer (configured in `Program.cs`). Login verifies the password with **BCrypt.Net-Next**, then issues a 7-day token carrying Name/Email/Role claims signed (HS256) with `Jwt:Key`. Note: `[Authorize]` is **not** broadly applied yet — most endpoints are currently open; tighten with `[Authorize(Roles = …)]` when adding protected operations.

### Core business logic — double counting (in `InventaireController`)
The system reconciles **two independent counts** per product (`ProduitInventaire`):
- `QuantiteComptage1` / `QuantiteComptage2` — filled by the two assigned supervisors; `modifier-quantite` routes the increment to count 1 or 2 based on the logged-in user.
- `EcartTheorique = Comptage1 − QuantiteTheorique` (vs. expected stock).
- `EcartPhysique = Comptage1 − Comptage2` (between the two counts).
- `QuantiteFinale` and `Statut` ("validé" / "écart" / "en attente") are derived from agreement between the two counts.
- A product is matched by `Ref` (+ optional `Gamme1` / `NumLot`); rows are auto-created if not pre-imported.

### Excel (`EPPlus` + `NPOI`)
- **Import** (`POST /api/inventaire/import`): reads `.xls` (NPOI `HSSFWorkbook`) / `.xlsx` (`XSSFWorkbook`); fixed column order (Ref, Designation, Famille, Gamme1, Gamme2, Nserie, Quantite, PrixUnitaire, MontantTotal, SuiviStock, Depot, DatePeremption). Validates depot, seeds `ProduitInventaire` with `QuantiteTheorique`, assigns the inventory's two users as Superviseur1/2.
- **Reports**: `rapportExcel` produces the full reconciliation sheet (écarts, statut, justification).
- EPPlus requires the `NonCommercial` license context (set in `appsettings.json` and `.csproj`).

### CORS
`Program.cs` defines `AllowAllOrigins` (permissive, currently active) and a stricter `AllowSpecificOrigin` for `http://localhost:4200`. Switch to the specific policy before production.

## Conventions
- Keep new application logic in the app-owned entities; treat `F_…` SAGE entities as scaffolded/read-mostly.
- Group new `InventaireController` actions by concern (counting / reports / affectation) — it already aggregates several backend concerns.
- DTOs live in `DTO/`; map entities → DTOs for API responses rather than returning EF entities directly.
